# 🔄 ゆるスケ

**崩れた勉強計画を自動で引き直す、大学の学期用スタディプランナー。**

👉 **使う**: https://shimi-works.github.io/yuru-suke/ （スマホなら「ホーム画面に追加」でアプリとして使えます）

計画アプリが挫折する最大の理由——「初日に立てた計画が3日目に現実と合わなくなり、直すのが面倒で開かなくなる」——だけを解決します。記録のアプリ（Studyplus等）とは土俵が違う、**再計画のアプリ**です。

## 特徴

- **時間の見積もりは不要**: 既定は**ただのタスク管理**。やることを書くだけで、1日◯件ずつ締切から逆算して自動で配られる。「これ何分かかる？」を考えなくていい（時間で管理したい人は設定でONにできる）
- **再計画はボタンではなく常時**: 開いた瞬間に、昨日までの未消化が吸収された「今日やること」が出ている。繰越ボタンはそもそも存在しない
- **やることは具体で**: 締切の下に「第3章 p45-60」のようなやることをぶら下げてチェック。**まとめて追加**や**クイック追加**で複数行を一気に登録
- **レポートは工程テンプレ**: レポート締切を作ると「資料集め→構成→本文→見直し」の工程が入った状態から始められる
- **サボり前提**: 「今日は無理 🫠」で今日ぶんを丸ごと明日以降へ。「→明日」で1件だけ後回しにもできる
- **正直**: 間に合わない締切は誤魔化さず警告し、何を削るべきかまで提示
- **カレンダーに書き出せる**: 締切と今日以降の予定を `.ics` でエクスポート → Googleカレンダー等に取り込み（片方向）
- **完全ローカル**: アカウント不要・サーバーなし。データは端末のlocalStorageのみ。JSONでバックアップ/引っ越し
- **AIはおまけで本体はアルゴリズム**: Gemini（無料枠・任意）に状態を渡して「引き直して」と相談できるが、キーなしでも全機能が動く

## 使い方

上のURLを開くだけ（ローカルなら `index.html` を直接開いても動きます）。科目名と締切日を入れれば、その日から「今日やること」の計算が始まります。締切の下に「やること」を足すほど、今日の指示が具体的になります（あとからで大丈夫）。

## 2つのモード

| | タスクモード（既定） | 時間モード（設定でON） |
|---|---|---|
| 入力 | やることを書くだけ | やることに分を見積もる |
| 配分の単位 | 1日◯件 | 1日◯分 |
| 分割 | しない（1件まるごと） | 長いものは複数日に自動分割 |

「勉強時間の記録」で戦わないのがこのアプリの立場です（そこはStudyplusの土俵）。だから既定では時間を一切扱わず、**振り返りもタスクの完遂ベース**（完了率・締切に間に合ったか・連続日数）で見ます。

## 開発

- 再配分エンジン（EDF貪欲割当＋余裕度警告）は純関数として `index.html` 内のマーカーブロックに隔離。2モードは「コスト」の抽象化で1つのアルゴリズムに統一
- テスト: `node tests/planner.test.mjs`

## アーキテクチャの芯

状態（JSON）とエンジン（再配分の純関数）を分離し、AIは「状態を読んで提案するだけの交換可能なエンジン」に徹する設計。AIの応答はスキーマ検証→許可フィールド制限→差分プレビュー承認の3段ガードを通り、タスクの完了状態や締切日は書き換えられません。
